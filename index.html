02

Is IDF right for your team?
Good fit
  • ✓Your team uses AI coding agents — Claude Code, Cursor, Copilot Workspace — as the primary way work gets built
  • ✓You ship continuously, not in quarterly releases
  • ✓A human Product Owner can own the gate — reading a short report and approving each release
  • ✓You want to replace sprint ceremonies with a lighter, outcome-driven loop
  • ✓Team size: 1–15 people
Not a fit — yet
  • —Your team uses AI only for autocomplete or code review assistance — execution is still fully human
  • —You're in a regulated release environment where monthly or quarterly cycles are a hard constraint
  • —You need a framework that manages dozens of squads and cross-programme dependencies at scale
  • —You're looking for a code architecture pattern or a routing library — IDF is a team process, not a software pattern
